//  순환 참조 발생시 Application 동작 시점과 Runtime 시점 중 필드주입은 RunTime 시점에 해당하기 때문에
//  메모리에 계속 할당되고 StackOverFlow가 발생했다. 하지만 2.6버전 이상부터는 Application 동작 시점에서 잡힌다.
//  Spring이 아니면 해당 필드에 주입할 수 있는 방법이 없다.
//  단위테스트에서는 필드주입만 사용

	Restaurant, Chef 클래스 선언
	단위 테스트에서 주입 확인

	School, Teacher, Student 클래스 선언
	단위 테스트에서 주입 확인

Qualifier 실습
	Resturant.java, Outback.java, Vips.java 세 객체를 선언한 뒤 상속관계를 판단하여 하나의 객체를 인터페이스로 선언한다.
	각 레스토랑에 셀바 이용 가능 여부와 스테이크 가격을 필드로 구성한다. 스테이크 가격은 항상 똑같지만 각 레스토랑에서 변경 가능하다.
	기본 레스토랑은 아웃백로 설정한다.
